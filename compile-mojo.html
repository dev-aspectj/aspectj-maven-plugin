aspectj:compile

Full name:

dev.aspectj:aspectj-maven-plugin:1.14:compile

Description:

Weaves all main classes. AspectJ Compiler Plugin.

Attributes:

  • Requires a Maven project to be executed.
  • Requires dependency resolution of artifacts in scope: compile.
  • The goal is thread-safe and supports parallel builds.
  • Binds by default to the lifecycle phase: compile.

This option can be used in case you want to use AJC options not (yet) supported by this plugin.

Caveat: Be careful when using this option and select the additional compiler arguments wisely, because behaviour is undefined if you add arguments which have already been added by the plugin using regular parameters or their default values. Caveat: If you configure a common directory, e.g. ${project.build.directory}/aj-build, in the plugin configuration, please make sure to choose different argumentFileNames for compile vs test-compile goal execution configurations to avoid one file overwriting the other.

The file is also used to determine if recompilation is necessary in subsequent builds. The argumentFileDirectory can be changed, which is helpful for reproducible builds.

Use this option in order to make sure that not only the generated byte code conforms to the target level but also the code is compiled against a specific JDK's API.

  </Xset>
